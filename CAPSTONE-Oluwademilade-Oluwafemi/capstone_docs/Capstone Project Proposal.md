# Capstone Project Proposal Summary

* * * * *

## Project Overview and Problem

My capstone project focuses on building a *predictive decision-support system* that analyzes life expectancy across regions in the United States using publicly available government data. Life expectancy varies widely due to factors such as income, education, population density, and access to healthcare. While large datasets exist, there is no simple system that integrates this data into a form that is easily interpretable for analysis and decision-making.

This project addresses that gap by developing a working prototype that combines data, analytics, and system design to predict life expectancy and highlight the key factors influencing these outcomes.

* * * * *

Prototype and Curricular Area Synthesis
---------------------------------------

The prototype will be a simple web-based dashboard application that allows a user to select a region (e.g., state or county) and view:

-   Predicted life expectancy

-   Risk classification (e.g., low, medium, high)

-   Key contributing factors

The prototype demonstrates the integration of three primary CISBA curricular areas:

-   **Software Systems (SS):** Development of a functional application using Python and a web-based interface

-   **Data Analytics (DA):** Application of predictive modeling techniques (e.g., regression,) to analyze life expectancy

-   **Data Management (DM):** Collection, cleaning, and integration of multiple government datasets into a structured format

* * * * *

Paper and Full Curricular Integration
-------------------------------------

The synthesis paper will expand on the prototype by incorporating all four CISBA areas:

-   **Software Systems (SS):** System design and implementation of the application

-   **Data Analytics (DA):** Modeling approach, analysis, and interpretation of results

-   **Data Management (DM):** Data sourcing, preparation, and integration processes

-   **Cybersecurity and Networking (CN):** Consideration of data privacy, ethical use of public health data, and potential risks associated with misuse or misinterpretation

The paper will also reference prior projects completed during the program (e.g., security audits, application development, and data analysis work) to demonstrate how this capstone represents the evolution and synthesis of my learning.

* * * * *

Feasibility and Requirements Alignment
--------------------------------------

This project is feasible within the semester timeframe because:

-   It uses publicly available datasets from sources such as the Centers for Disease Control and Prevention and the United States Census Bureau

-   It relies on tools and techniques already covered in the CISBA program

-   The prototype is designed to be simple, functional, and clearly demonstrable

The project will meet all capstone requirements by:

-   Producing a working prototype hosted in the WTAMU GitHub repository
